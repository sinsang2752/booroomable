# 게임 화면 레이아웃 개편 작업지시서

## 0. 이 문서의 목적

현재 게임 화면의 중앙 조작 카드가 보드와 타일을 가리는 문제를 해결한다.

이번 작업은 **게임 규칙이나 서버 로직을 바꾸는 작업이 아니다.** 기존 기능과 액션은 그대로 유지하고, 화면을 아래와 같은 두 영역으로 재배치하는 UI 개편이다.

- 왼쪽: 보드와 주사위가 중심이 되는 플레이 영역
- 오른쪽: 현재 턴, 플레이어 자산, 이벤트 메시지, 행동 버튼을 모은 턴 패널

작업 전 `CLAUDE.md`의 규칙을 다시 확인하고, 현재 컴포넌트와 상태 흐름을 먼저 파악한다.

---

## 1. 현재 화면에서 해결할 문제

1. 중앙의 큰 흰색 카드가 여러 타일과 보드 내부를 가린다.
2. 현재 턴, 주사위, 잔액, 메시지, 행동 버튼이 한 카드에 섞여 정보 우선순위가 불분명하다.
3. 매우 넓은 화면에 비해 중앙 일부에만 정보가 몰려 있다.
4. 행동 버튼이 게임판 위에 떠 있어 보드와 조작 영역의 경계가 모호하다.
5. 주사위가 단순 결과 표시처럼 보여 게임의 핵심 행동이라는 인상이 약하다.
6. 이벤트 메시지가 다른 정보 사이에 끼어 있어 쉽게 지나치게 된다.

---

## 2. 목표 화면 구조

```text
┌────────────────────────────────────────────────────────────────────┐
│  ┌───────────────────────────────────────────┐ ┌─────────────────┐ │
│  │ 타일                                 타일 │ │ 현재 턴   타이머│ │
│  │                                           │ │ 플레이어 정보   │ │
│  │                                           │ │                 │ │
│  │              주사위 애니메이션             │ │ 보유 자산        │ │
│  │                                           │ │ 지폐·돈다발 이미지│ │
│  │                                           │ │                 │ │
│  │ 타일                                 타일 │ │ 이벤트 메시지    │ │
│  │             채팅 입력 영역                │ │ [주 행동 버튼]   │ │
│  └───────────────────────────────────────────┘ └─────────────────┘ │
└────────────────────────────────────────────────────────────────────┘
```

핵심 원칙은 **보드 위에 큰 카드나 모달을 상시 겹치지 않는 것**이다. 주사위 애니메이션과 플레이어 말풍선처럼 게임판에 속하는 요소만 보드 영역 안에 표시한다.

---

## 3. 전체 레이아웃

### 3.1 기본 배치

- 게임 화면을 하나의 가로형 프레임으로 구성한다.
- 프레임 내부는 `플레이 영역 + 턴 패널`의 2열 구조다.
- 바깥 여백: 16px
- 두 영역 사이 간격: 16px
- 턴 패널 너비: 최소 280px, 권장 300~320px, 최대 340px
- 플레이 영역: 남는 너비를 모두 사용
- 전체 높이: 현재 데스크톱 하단 위젯 성격을 유지하며 약 360~440px 범위
- 화면 전체에 세로 스크롤이 생기지 않게 한다.
- 브라우저 상단 UI는 게임 레이아웃 크기 계산에서 제외한다.

### 3.2 컨테이너 표현

- 전체 배경은 따뜻한 아이보리 또는 매우 옅은 회색을 사용한다.
- 플레이 영역과 턴 패널은 각각 독립된 카드처럼 보이게 한다.
- 카드 테두리는 강한 검정선 대신 옅은 회보라색 또는 중성 회색을 사용한다.
- 모서리는 20~28px 정도로 둥글게 한다.
- 그림자는 매우 약하게 사용하고, 깊은 드롭 섀도나 유리 효과는 사용하지 않는다.
- 현재의 밝고 부드러운 인상은 유지하되 정보 대비는 더 선명하게 만든다.

## 4. 왼쪽 플레이 영역

### 4.1 보드

- 기존 40칸 구조를 유지한다.
  - 위 19칸
  - 아래 19칸
  - 왼쪽 코너 `출발`
  - 오른쪽 코너 `무인도`
- 기존 알약형 트랙의 정체성을 유지하되, 플레이 영역 안에 완전히 들어오도록 크기를 다시 계산한다.
- 위·아래 타일의 크기와 간격은 동일해야 한다.
- 중앙 턴 카드가 사라진 뒤에도 타일이 불필요하게 커지지 않도록 한다.
- 타일 이름, 가격, 건물 상태가 한눈에 읽히는 최소 글자 크기를 유지한다.
- 황금열쇠, 우주여행, 사회복지기금 등 특수칸은 색으로 구분하되 전체 팔레트에서 튀지 않게 한다.
- 출발과 무인도는 일반 타일보다 약간 크게 보여도 된다.

### 4.2 플레이어 말과 잔액

- 각 플레이어 말은 현재 말 색상을 유지한다.
- `CLAUDE.md` 규칙대로 말 주변의 잔액 배지는 항상 표시한다.
- 여러 플레이어가 같은 칸에 있을 때 겹쳐서 한 명이 사라지지 않도록 작은 부채꼴 또는 가로 스택으로 배치한다.
- 현재 턴 플레이어의 말에는 다음 중 하나로 강조 효과를 준다.
  - 얇은 외곽 링
  - 부드러운 펄스
  - 작은 `TURN` 표시
- 강조 애니메이션은 타일 텍스트를 가리지 않아야 한다.

### 4.3 중앙 주사위 스테이지

- 보드 중앙에는 큰 흰색 카드 대신 주사위 전용 스테이지를 둔다.
- 기본 상태에서는 주사위 2개와 짧은 안내 문구만 표시한다.
- 주사위는 화면 중심보다 살짝 위에 배치해 아래 채팅 영역과 시각적으로 분리한다.
- 주사위가 굴러가는 동안에는 700~900ms 정도의 짧은 애니메이션을 사용한다.
- 애니메이션 중에도 타일이나 턴 패널의 위치가 움직이면 안 된다.
- 결과가 나온 뒤에는 두 주사위 눈과 합계를 함께 읽을 수 있게 한다.
- 더블이면 `더블! 한 번 더`와 같은 짧은 강조 문구를 주사위 아래에 표시한다.
- 주사위를 정적인 래스터 이미지 한 장으로 만들지 않는다. 회전과 눈 변경이 필요한 요소이므로 CSS, SVG 또는 현재 애니메이션 방식을 유지한다.

### 4.4 선택 단계

출발칸 보너스와 우주여행처럼 보드에서 칸을 골라야 하는 상태는 기존 `selectableTiles`와 `onSelectTile` 흐름을 그대로 사용한다.

- 선택 가능한 칸: 플레이어 색상의 선명한 테두리와 부드러운 펄스
- 선택 불가능한 칸: 평소 표현 유지
- 마우스 오버: 다음 등급, 건설비 또는 이동 목적지 정보를 작은 툴팁으로 표시
- 보드 중앙에는 `업그레이드할 땅을 선택하세요`처럼 한 줄 안내만 표시
- 큰 선택 모달을 다시 보드 위에 띄우지 않는다.
- `건너뛰기` 또는 `이동 안 함`은 오른쪽 패널의 보조 행동으로 표시한다.

### 4.5 채팅

채팅 기능을 삭제하지 않는다.

- 플레이 영역 하단 중앙에 한 줄 입력창을 둔다.
- 타일과 겹치지 않도록 보드 내부의 빈 중앙 하단 또는 플레이 영역의 하단 슬롯에 배치한다.
- 최대 20자 제한을 유지한다.
- 채팅 전송 시 기존처럼 플레이어 말 위에 3.5초 동안 말풍선을 표시한다.
- 이벤트 메시지와 채팅 메시지를 같은 영역에 섞지 않는다.
- 현재 채팅은 저장되지 않는 Broadcast 방식이므로 존재하지 않는 `로그 열기` 기능을 새로 만들지 않는다.

## 5. 오른쪽 턴 패널

턴 패널은 매번 위치가 바뀌는 모달이 아니라 화면에 고정된 게임 대시보드다.

### 5.1 상단: 현재 턴

- 왼쪽에 `현재 턴` 레이블
- 오른쪽에 남은 시간
- 그 아래에 현재 플레이어의 말 색상, 닉네임, 턴 번호를 표시
- 남은 시간이 10초 이하가 되면 숫자 또는 타이머 링을 경고색으로 바꾼다.
- 경고 상태에서도 패널 전체가 깜빡이게 하지 않는다.

권장 정보 순서:

```text
현재 턴                         18턴
[파란 말] 님                    10초
```

### 5.2 중단: 보유 자산

- 제목은 `보유 자산`
- 큰 금액을 가장 중요한 정보로 표시한다.
- 금액 옆 또는 아래에 지폐·돈다발 일러스트를 배치한다.
- 현재 턴 플레이어의 자산을 보여준다.
- 내 자산이 별도로 필요하다면 패널 안에 같은 크기로 중복 표시하지 말고, 기존 말의 잔액 배지를 통해 확인하게 한다.
- 숫자는 `1,520`처럼 천 단위 구분을 적용한다.
- 통행료 지불이나 상금 수령 시 금액 변화는 짧은 증가·감소 표시로 보여준다.
  - 수입: `+200`
  - 지출: `-100`
- 지나친 숫자 카운팅 애니메이션은 사용하지 않는다.

### 5.3 이벤트 메시지

- 자산 영역 아래에 독립된 메시지 박스를 둔다.
- 서버의 기존 `notice` 또는 현재 화면의 이벤트 안내 문구를 이곳으로 옮긴다.
- 한 번에 가장 최근의 중요한 메시지 하나를 보여준다.
- 최대 2~3줄까지만 표시하고 긴 문장은 말줄임 처리하거나 작은 상세 보기로 확장한다.
- 메시지 종류를 아이콘과 옅은 배경색으로 구분한다.
  - 일반 안내: 중성색
  - 수입/보상: 녹색 계열
  - 지출/통행료: 주황색 계열
  - 파산/경고: 붉은색 계열
  - 황금열쇠: 금색 계열
- 이 영역은 채팅창이 아니다.

### 5.4 하단: 행동 버튼

- 패널 하단에 현재 단계의 가장 중요한 행동 하나를 큰 버튼으로 표시한다.
- 버튼 위치는 상태가 바뀌어도 움직이지 않게 패널 하단에 고정한다.
- 현재 플레이어 색상을 기본 강조색으로 사용할 수 있지만, 텍스트 대비를 우선한다.

상태별 예:

| 게임 상태 | 주 행동 | 보조 행동 |
|---|---|---|
| `awaiting-roll` | 주사위 굴리기 | 없음 |
| `awaiting-purchase-decision` | 구매하기 | 건너뛰기 |
| `awaiting-build-decision` | 건축하기 | 건너뛰기 |
| `awaiting-initial-build-decision` | 다음 등급 건축 | 건너뛰기 |
| `awaiting-start-bonus-build` | 선택한 땅 건축 | 건너뛰기 |
| `awaiting-space-travel-destination` | 선택한 칸으로 이동 | 이동 안 함 |
| 다른 플레이어의 턴 | `OO님 차례입니다` | 버튼 없음 |
| 서버 처리 중 | 처리 중… | 중복 클릭 불가 |

- 구매나 건축 단계에서 필요한 비용을 버튼 안이나 바로 위에 표시한다.
- 내 턴이 아닐 때 비활성화된 굴리기 버튼을 보여주지 않는다.
- 타임아웃 자동진행 규칙은 변경하지 않는다.
- `게임 포기하기`는 주 행동과 경쟁하지 않도록 작은 텍스트 버튼으로 패널 맨 아래 또는 전체 프레임 하단에 둔다.
- 게임 포기 확인창은 기존 동작을 유지한다.

## 6. 시각 디자인 방향

### 6.1 분위기

- 가볍고 친근한 캐주얼 보드게임
- 장시간 화면 아래에 띄워두어도 피로하지 않은 밝은 톤
- 지나치게 유아적이거나 카지노처럼 보이는 표현은 피한다.
- 현재 화면의 단정한 인상을 살리면서 게임 요소의 생동감을 조금 더한다.

### 6.2 색상

- 배경: 따뜻한 아이보리
- 기본 카드: 흰색
- 기본 테두리: 옅은 회보라 또는 중성 회색
- 본문 글자: 짙은 청회색
- 보조 글자: 중간 회색
- 핵심 행동: 현재 플레이어 색상 또는 기존 파란색
- 위험 행동: 빨간색을 면 채움으로 남용하지 말고 텍스트·테두리 중심으로 사용

플레이어 색상 `blue / red / green / amber`는 게임 상태를 표현하는 값이므로 임의로 없애거나 서로 비슷한 색으로 바꾸지 않는다.

### 6.3 타이포그래피

- 게임 전체에서 한글 가독성이 좋은 산세리프 계열을 유지한다.
- 금액과 타이머는 숫자 폭이 안정적인 글꼴 설정을 사용한다.
- 제목, 금액, 본문, 보조 설명의 네 단계만 사용한다.
- 지나치게 다양한 폰트 굵기를 사용하지 않는다.

### 6.4 아이콘

- 간단한 UI 아이콘은 생성형 이미지로 만들지 않는다.
- 타이머, 알림, 채팅, 건너뛰기 등의 아이콘은 동일한 SVG 아이콘 체계로 통일한다.
- 이모지는 임시 표현으로만 허용하고 최종 UI의 핵심 아이콘으로 사용하지 않는다.

## 7. 생성형 AI 에셋 지시

이번 구조에서 우선 필요한 생성형 에셋은 **보유 자산 영역의 지폐·돈다발 일러스트 1종**이다.

### 7.1 에셋 사양

- 파일명 예정: `cash-stack.png`
- 원본 크기: 1024×1024px
- 표시 크기: 약 88~120px
- 배경: 완전 투명
- 구도: 정면보다 약간 위에서 바라본 3/4 시점
- 형태: 지폐 묶음 2~3개와 동전 몇 개
- 스타일: 부드러운 3D 클레이 또는 정교한 캐주얼 게임 아이콘
- 색상: 민트, 크림, 골드 중심
- 외곽선: 두껍지 않게
- 그림자: 에셋 내부에 약한 접지 그림자만 포함
- 금지: 실제 국가 화폐, 인물 초상, 로고, 읽을 수 있는 문자, 워터마크, 과도한 광택

### 7.2 적용 전 규칙

- 최종 에셋은 Codex가 별도로 생성해 전달한다.
- 에셋 파일이 전달되기 전까지 Claude가 인터넷에서 임의 이미지를 내려받지 않는다.
- 임시 구현이 필요하면 동일한 크기의 중성 플레이스홀더를 사용한다.
- 에셋이 없어도 금액 정보와 레이아웃은 정상적으로 작동해야 한다.

주사위는 상태에 따라 눈과 회전이 바뀌어야 하므로 생성형 이미지 에셋 대상으로 삼지 않는다.

## 8. 반드시 유지할 기존 기능

- 40칸 보드와 타일 인덱스
- 실시간 플레이어 위치와 잔액
- 현재 턴과 남은 시간
- 주사위 굴리기 및 더블 처리
- 구매, 건축, 건너뛰기
- 출발칸 보너스용 타일 선택
- 우주여행 목적지 선택
- 채팅 Broadcast 및 말풍선
- 게임 포기하기
- 서버의 `notice`
- 타임아웃 자동진행
- 내 턴이 아닐 때 조작 방지
- Supabase 상태 동기화

UI 개편을 이유로 게임 엔진, DB 스키마, Edge Function, 액션 이름, 자동 기본값을 변경하지 않는다.

## 9. 작업 순서

한 번에 전체 스타일을 갈아엎지 말고 다음 순서로 진행한다.

1. 현재 게임 화면 컴포넌트와 상태별 버튼 분기를 확인한다.
2. 중앙 카드의 내용을 `플레이 영역`과 `턴 패널`로 분리한다.
3. 보드 크기와 40개 타일 배치를 새 왼쪽 영역에 맞춘다.
4. 오른쪽 패널에 현재 턴, 자산, 이벤트 메시지, 행동 버튼을 연결한다.
5. 채팅 입력창을 타일과 겹치지 않는 왼쪽 하단 영역으로 이동한다.
6. 선택 가능한 타일과 주사위 애니메이션의 위치를 조정한다.
7. 상태별 화면을 하나씩 확인한다.
8. 마지막에 색상, 간격, 그림자, 애니메이션을 다듬는다.

각 단계가 끝날 때마다 앱이 실행되고 실제 게임 진행이 가능한 상태를 유지한다.

## 10. 상태별 확인 목록

최소한 아래 장면을 직접 확인한다.

1. 내 차례이고 주사위를 굴리기 전
2. 다른 플레이어 차례
3. 주사위가 굴러가는 중
4. 주사위 결과가 표시된 상태
5. 구매 여부를 선택하는 상태
6. 재방문 건축 선택 상태
7. 최초 구매 후 반복 건축 상태
8. 출발칸 보너스로 보드에서 땅을 선택하는 상태
9. 우주여행 목적지를 선택하는 상태
10. 황금열쇠 또는 사회복지기금 메시지가 나온 상태
11. 남은 시간이 10초 이하인 상태
12. 플레이어가 같은 칸에 2명 이상 겹친 상태
13. 파산한 플레이어가 있는 상태
14. 게임 종료 상태

## 11. 완료 조건

- 중앙의 큰 조작 카드가 사라져 모든 타일이 항상 보인다.
- 화면이 왼쪽 플레이 영역과 오른쪽 턴 패널로 명확히 나뉜다.
- 40개 타일이 잘리거나 서로 겹치지 않는다.
- 주사위 애니메이션이 보드 중앙에서 명확하게 보인다.
- 현재 턴, 플레이어, 자산, 남은 시간이 오른쪽 패널에서 즉시 읽힌다.
- 이벤트 메시지와 채팅 메시지가 분리되어 있다.
- 모든 선택 단계에서 주 행동과 보조 행동을 찾기 쉽다.
- 다른 사람 턴에는 조작 가능한 버튼이 보이지 않는다.
- 기존 게임 규칙, 서버 동기화, 자동진행 동작이 바뀌지 않는다.
- 지폐 에셋이 없어도 레이아웃이 깨지지 않는다.
- 대상 데스크톱 해상도에서 세로 스크롤이 생기지 않는다.
